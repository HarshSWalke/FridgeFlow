FridgeFlow – Complete Technical Specification (Page by Page)

Global Architecture FirstBefore pages, understand these building blocks:Data Models (what gets stored)

User → profile + vendor contacts + preferencesItem → name, category, quantity, unit, threshold, dateAdded, expiryDate, pricePerUnit, addedByOrder → vendorId, items[], status, dateSent, messageTextTransaction → itemId, quantity, pricePerUnit, date, type (added/consumed/expired)

Core Logic Rules

Every item has a threshold quantity — when current quantity ≤ threshold, item auto-flags as "Low"Every item tracks quantity as a number — not just "present/absent"Expired = today's date > expiryDateReorder list = union of (low stock items + manually added items + finished items)

Item GridItems displayed as cards in a responsive grid (3 columns desktop, 2 on tablet, 1 on mobile).Each Item Card shows:

Item name (bold)Colored freshness dot (🟢🟡🔴) — top right corner of cardQuantity + unit (e.g., "1.5 L")Expiry date (e.g., "Expires Jun 5" or "No Expiry")Date added (small text, grey)Category tag (small pill badge)Two action buttons: Edit ✏️ | Mark Finished ✓Three-dot menu (…): Edit | Add to Reorder | Delete

Logic behind quantity:

Stored as a float (e.g., 1.5, 500, 12)Unit stored separately (L, kg, g, pcs, packets)When user edits quantity, they can type new value OR use +/– stepper buttonsQuantity can never go below 0 — if user sets 0, system prompts "Mark as Finished?"

Freshness color logic:

No expiry set → always 🟢expiryDate > today + 3 days → 🟢expiryDate within today to today + 3 days → 🟡expiryDate < today → 🔴

Add Item Flow (triggered by + Add Item button)Opens a side drawer (slides in from right) — not a new page, not a popup modal.Fields in the drawer:

Item Name — text input with autocomplete (suggests from user's history + common items list)Category — dropdown (Dairy / Vegetables / Fruits / Dry Goods / Beverages / Snacks / Leftovers / Other)Quantity — number inputUnit — dropdown (kg / g / L / mL / pcs / packets / bottles)Date Added — auto-fills today, user can changeExpiry Date — date picker. Toggle: "This item has no fixed expiry" (hides the date picker)Price per unit — number input (optional, used for spending tracker). Placeholder: "e.g. ₹52"Threshold quantity — number input. Helper text: "Alert me when below this amount". Pre-fills based on signup defaults if item was pre-configured.Assign Vendor — dropdown of saved vendors (optional — used to auto-assign reorders)

Duplicate detection logic:

On item name input, check if same item name already exists in same categoryIf yes → show inline warning: "Milk already exists in Dairy (1.5L). Want to update quantity instead?"Two options: Update Existing | Add as New (different brand/type)

Edit Item:Same drawer opens pre-filled with existing values. All fields editable. Save overwrites. Cancel discards changes.

Mark as Finished:

One-click button on cardSets quantity to 0Creates activity log entryChecks if item has a threshold → if yes, auto-adds to Reorder section with a toast notification: "Paneer added to your Reorder list"
